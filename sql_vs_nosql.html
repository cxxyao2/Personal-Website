CAP theorem:

For distributed computing system, it is impossible to satisfy 3 requirements at the same time:

  • Consistency
  • Availability
  • Partition tolerance
Scalable.Efficient.Available.
ACID Principles
  • Atomicity: an indivisible and irreducible series of database operations such that either all occur, or nothing occurs. eg. 1) withdraw $10 from A account 2) save $10 to B account. 1), 2) all are committed or rollback.
  • Consistency:Data written to DB must be valid according to all defined rules,including constraints, trigger, cascades, and other combinations thereof2.
  • Isolation: parallel transactions are isolated. B transaction cannot see the dirty data(not committed) of A transaction
  • Durability: data persistance after server is shutdown
