Most Commonly Used RxJS Operators

  • Cold Observable vs Hot Observable
    • cold:  of, from
    • hot:  subject, publish, connect, refCount, publicReplay
  • Four Subjects
    • Subject
    • BehaviorSubject
    • AsyncSubject
    • ReplaySubject
  • Maps  
    • map
    • SwitchMap
      1. It is a flattening operator. It maps an Observable to a new Observable and emit values.
      2. It can cancel in-flight network requests
      3. It allows only one active inner subscription
      4. It only map the most recent value at a time to reduce unnecessary output( and network traffic).
        // someComponent.ts
  ngOnInit() {
        this.validSearch$ = this.onSearchUser$
            .pipe(
                debounceTime(1000),
                map(event => (event.target).value),
                distinctUntilChanged(),
                filter(input => input !== ""),
                switchMap(data => this.rxjsSearchableInputService.searchUser(data))
            )
}
    • mapTo
      • map everyting to a constant value
      const source = interval(2000);
const example = source.pipe(mapTo(‘Hello World’));
const subscribe = example.subscribe(val => console.log(val)) ;
// output: Hello World
    • MergeMap( FlatMap )
      1. It is an flattening operator
      2. If every request needs to complete, it is the correct options
      3. It allows multiple active inner subscriptions
    • concatMap
      • use it if the order of emission and subscription of inner observables is important,
